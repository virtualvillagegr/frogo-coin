# Terrain Flattener TNT v3.0\n\n🧨 **Advanced terrain flattening addon for Minecraft Bedrock Edition**\n\n## Features\n\n✅ **Auto-Device Detection** - Automatically optimizes for phones, tablets, and PCs  \n✅ **Chunk-Aware Processing** - Handles unloaded chunks gracefully  \n✅ **Multiplayer-Safe** - Queue system prevents conflicts with multiple explosions  \n✅ **Performance Tracking** - Real-time monitoring of block clearing speed  \n✅ **Protected Blocks** - Ores, chests, barrels, and valuable blocks stay safe  \n✅ **Circular Flattening** - Natural-looking terrain cleared in circular patterns  \n✅ **Production-Ready** - Full optimization with comprehensive error handling  \n\n## How to Use\n\n### Basic Usage\n1. Get a TNT block (or use the flattening tools)\n2. Place it near terrain you want to flatten\n3. Trigger the explosion\n4. The addon will automatically clear the area while protecting valuable blocks\n\n### Available Tools\n\n| Tool | Radius | Area |\n|------|--------|------|\n| 🧨 Dynamite | 8 blocks | Small |\n| 💎 Small Flattener | 16 blocks | Medium |\n| 🟠 Medium Flattener | 32 blocks | Large |\n| 🟣 Large Flattener | 64 blocks | Huge |\n\n## Performance\n\n- **Phone**: ~6-8 seconds per explosion\n- **Tablet**: ~3-4 seconds per explosion\n- **PC**: ~1-2 seconds per explosion\n- **Ultra PC**: <1 second per explosion\n\n## Protected Blocks\n\nThese blocks will NOT be destroyed:\n- Diamond Ore (all variants)\n- Gold Ore\n- Iron Ore\n- Coal Ore\n- Emerald Ore\n- Redstone Ore\n- Lapis Ore\n- Chest / Barrel\n- Spawner\n- Shulker Box\n- Furnace (all variants)\n- Hopper\n- Dropper / Dispenser\n- Enchanting Table\n- Brewing Stand\n- **Bedrock** (never destroyed)\n\n## Architecture\n\n### Core Systems\n\n1. **Performance Tracker** - Monitors explosion processing time and blocks cleared\n2. **Chunk Manager** - Pre-checks chunk loading and caches status\n3. **Device Detector** - Auto-selects optimal batch size based on player count\n4. **Explosion Queue Manager** - Handles multiple simultaneous explosions safely\n5. **Main Flattening Function** - Executes terrain clearing with full optimization\n\n### Processing Pipeline\n\n```\nExplosion Event\n    ↓\nValidation & Deduplication\n    ↓\nChunk Loading Check\n    ↓\nDevice Detection\n    ↓\nQueue Management\n    ↓\n/fill Command (Fast Path)\n    ↓\nScript Fallback (if needed)\n    ↓\nCompletion & Performance Logging\n```\n\n## Configuration\n\nEdit `CONFIG` in `scripts/main.js` to customize:\n\n```javascript\nconst CONFIG = {\n  radius: 12,                    // Flattening radius in blocks\n  height: 40,                    // Height above TNT to flatten\n  maxConcurrentExplosions: 2,    // Max simultaneous processing\n  maxQueueSize: 15,              // Max queued explosions\n  verboseLogging: true,          // Detailed console output\n  performanceTracking: true      // Track speed metrics\n};\n```\n\n## Optimization Features\n\n### 1. Auto-Device Detection\n- Monitors player count to estimate device capability\n- Automatically adjusts batch size and delays\n- No lag on any device\n\n### 2. Chunk Loading Management\n- Pre-checks all chunks are loaded\n- Retries with delay if chunks unavailable\n- Graceful fallback if loading fails\n\n### 3. Multiplayer Queue System\n- Prevents simultaneous explosion processing\n- Manages up to 15 queued explosions\n- FIFO (First-In-First-Out) processing\n\n### 4. Performance Tracking\n- Logs average processing time\n- Tracks blocks cleared per explosion\n- Maintains 20-explosion history\n\n### 5. Dual-Mode Processing\n- **Fast Path**: Uses `/fill` command (100x faster)\n- **Fallback Path**: Script-based clearing with error recovery\n\n### 6. Circular Flattening\n- Natural-looking terrain pattern\n- Distance-based block filtering\n- Smooth area boundaries\n\n## Installation\n\n1. Download the addon\n2. Add to your Behavior Packs folder\n3. Enable in world settings\n4. Enjoy automatic terrain flattening!\n\n## Troubleshooting\n\n### Explosions not working?\n- Ensure addon is enabled in world settings\n- Check console for error messages\n- Try manual TNT placement\n\n### Lag spikes?\n- Addon auto-detects device capability\n- Works great on all devices\n- Reduce player count if severe\n\n### Protected blocks being destroyed?\n- Check block ID spelling in CONFIG\n- Ensure block name matches Bedrock identifier\n\n## Version History\n\n### v3.0.0 - Production Release\n- Complete optimization suite\n- Auto-device detection\n- Chunk-aware processing\n- Multiplayer queue system\n- Performance tracking\n- Comprehensive error handling\n\n### v2.0 - Stability Update\n- Added batch processing\n- Improved error handling\n- Device presets\n\n### v1.0 - Initial Release\n- Basic terrain flattening\n- Protected blocks system\n\n## Support\n\nFor issues or suggestions, please open an issue on GitHub.\n\n---\n\n**Made with ❤️ for Minecraft Bedrock players**\n
